# Feature Specifications

機能仕様は、実装前に合意する振る舞いの正本です。機能ごとに
`feature-spec.template.md` をコピーし、`<feature-name>.md` として保存します。

## Definition of Ready

計画へ進むには、次の条件をすべて満たします。

- [ ] 対象ユーザーとシナリオが明確である。
- [ ] 各要求が一意な `REQ-###` を持つ。
- [ ] 各要求に観察可能な `AC-###` がある。
- [ ] 正常系、境界、失敗時の振る舞いが定義されている。
- [ ] 非機能境界、依存、out of scope が明記されている。
- [ ] 振る舞いを変え得る未解決事項が残っていない。
- [ ] Status が `Approved` で、承認者と日付が記録されている。

Ready でない仕様から実装計画を作らないでください。探索的な調査が必要なら
[Experiment Log](../04_experiments.md) に登録します。

## Identifier Rules

- `REQ-###` はこの機能内で一意にし、承認後は再採番しません。
- `AC-###` は具体的な入力、操作、観察結果を記述します。
- 削除する要求は消去せず `Deprecated` とし、理由または後継を残します。
- 計画と検証報告は仕様への相対リンクと ID の両方を記載します。
